    Projeto API

    Esta API RESTful provê cadastro e autenticação de usuários e um endpoint de consulta que realiza scraping em tempo‑real das últimas 10 notícias do site Hacker News.

    https://news.ycombinator.com/

    Autores: Lucas Novais e Mariana Albuquerque

    Etapa 1: Construção da API e Dockerinzing

    1 ·  Link rápido

    • Projeto no GitHub: https://github.com/lucasnov/fastapi-with-postgree
    • Imagem no Docker Hub: https://hub.docker.com/r/lucasnov/projeto1

    2 ·  Endpoints

    Método Rota Descrição Autenticação
    POST /registrar Cria novo usuário ❌
    POST /login Retorna JWT ❌
    GET /consultar Scraping Hacker News (Top‑10) ✅ (Bearer)
    GET /health-check Status da API ❌

    OpenAPI

    A página Swagger é servida em
    http://localhost:8000/docs.

    4 · Segurança

    • JWT de 24 h.
    • Hash de senha bcrypt (passlib).
    • Variáveis sensíveis via .env.

    5 · Execução Local

    Requisitos mínimos testados
    Docker 20.10 ou superior (já inclui o Docker Compose V2 embutido no CLI).

    a.  Clonar o repositório

    git clone https://github.com/lucasnov/fastapi-with-postgree.git
cd fastapi-with-postgree       # todos os comandos abaixo rodam na raiz do projeto

    b.  Configurar variáveis de ambiente

    Copie o arquivo de exemplo e ajuste credenciais, se necessário:

    cp .env.example .env
# edite .env (POSTGRES_USER, POSTGRES_PASSWORD, JWT_SECRET_KEY, etc.)
    .env.example 
    POSTGRES_USER=projeto
POSTGRES_PASSWORD=S3cur3P@ss
POSTGRES_DB=projeto
POSTGRES_HOST=db
DATABASE_URL=postgresql://projeto:S3cur3P%40ss@db:5432/projeto
JWT_SECRET_KEY=my_super_secret_jwt_key

    c.  Subir a stack com Docker Compose

    # inicializa containers em segundo plano
docker compose up -d

    7 · Pipeline de Deploy

    Docker Hub:

    # Depois de logado no devido usuário

# Build
docker build -t lucasnov/projeto1:v1 .

# Push
docker push lucasnov/projeto1:v1
docker tag lucasnov/projeto1:v1 lucasnov/projeto1:latest
docker push lucasnov/projeto1:latest

    8 · compose.yaml final

    services:
  db:
    image: postgres:15
    restart: unless-stopped
    hostname: db
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-projeto}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-S3cur3P@ss}
      POSTGRES_DB: ${POSTGRES_DB:-projeto}
      DATABASE_URL: ${POSTGRES_HOST:-db}
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U $${POSTGRES_USER:-projeto} -d $${POSTGRES_DB:-projeto}"]
      interval: 5s
      timeout: 5s
      retries: 10
      start_period: 10s

  app:
    image: lucasnov/projeto1:v1
    ports:
      - "8000:8000"
    env_file:
      - .env
    ## depende do estado “healthy” do db
    depends_on:
      db:
        condition: service_healthy

    O compose.yaml está na raiz do repositório.

    Etapa 2: Deploy na AWS Lightsail

    O objetivo desta etapa foi publicar a mesma stack FastAPI + PostgreSQL na nuvem, utilizando Lightsail Container Service para a aplicação e Lightsail Managed Database para o banco.

    1 ·  Link rápido

    • Swagger da API: https://fastapi-service.n3a1c9dx898r2.us-east-1.cs.amazonlightsail.com/docs

    2 · Passo‑a‑passo de publicação

    sequenceDiagram
    autonumber
    participant Dev
    participant DockerHub
    participant Lightsail
    participant DB as "Lightsail DB"
    Dev->>DockerHub: docker push lucasnov/projeto1:v1
    Dev->>Lightsail: create container‑service\n(app lucasnov/projeto1:v1)
    Lightsail-->>Dev: Endpoint HTTPS
    Dev->>DB: create database fastapi-db
    Dev->>Lightsail: update env vars (DATABASE_URL…)
    Lightsail-->>Dev: Health ✔
    1. Imagem pronta no Docker Hub – lucasnov/projeto1:v1.
    2. Criação do container service
    3. Service name: fastapi-service
    4. Power: Micro 1 GB, US $ 10 (primeiros 90 dias free).
    5. Scale: 1 nó
    6. Public endpoint: porta 8000 com health‑check em /health-check.
    7. Criação do banco gerenciado
    8. Plan: Standard 1 GB, US $ 15 (primeiros 90 dias free).

    9. Public mode: on para permitir acesso externo.

    10. Variáveis de ambiente configuradas no container.

    11. Deploy → Save & deploy (ou aws lightsail create-container-service-deployment …).

    • Plano do banco: Standard 1 GB, 2 vCPU, 40 GB SSD (trial 90 dias).

    4 · Recursos alocados & custos

    Período gratuito de 90 dias – Tanto o plano Micro do Container Service (1 GB) quanto o plano Standard 1 GB do banco gerenciado têm isenção total nos três primeiros meses para a primeira instância de cada serviço. Enquanto essa franquia estiver ativa, não haverá cobranças mensais.

    Projeção para custos normais (fora do período gratuito)

    Item Plano Preço Qtde Sub‑total
    Container Service Micro (1 GB) US $ 10 1 nó US $ 10
    Database Standard 1 GB US $ 15 1 US $ 15
    Total (após 90 dias) US $ 25 / mês

    Projeção para escalas exigidas (≤ US $ 50)

    Réplicas Nós Micro Custo Container + DB 1 GB Total
    1 1 1 × 10 = 10 15 25
    5 5 5 × 10 = 50 15 65 ⚠
    10 10 10 × 10 = 100 15 115 🚫

    Para 5 nós dentro do limite basta trocar para Nano (US $ 7) → 5 × 7 + 15 = 50.
